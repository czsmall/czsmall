NGINX常用核心功能

一：核心功能-反向代理

反向代理是指通过Nginx服务器接收客户端的请求,然后将这些请求转发给后端的服务器（如Web服务器、应用服务器），最后将后端服务器的响应返回给客户端。

在这个过程中，客户端并不知道真正处理请求的后端服务器，从而隐藏了服务器的实际IP地址，增强了系统的安全性。

反向代理的配置示例

负载均衡模式,

Nginx 负载均衡提供了几种不同的策略（即“模式”或“算法”）来将客户端请求分配给后端服务器。这些负载均衡模式适用于不同的场景，可以帮助优化系统性能和可靠性。以下是 Nginx 支持的主要负载均衡模式：

1. 轮询（Round Robin）

  • 描述：这是 Nginx 的默认负载均衡模式。请求依次分发给每个后端服务器，按顺序循环处理请求。

  • 优点：简单且适用于大多数情况，负载分布较为均匀。

  • 适用场景：当各服务器的性能和负载处理能力相当时，使用轮询算法效果较好。

2. 最少连接数（Least Connections）

  • 描述：请求分发给当前活动连接数最少的服务器。

  • 优点：适合处理时间不均匀的请求，能够更好地平衡服务器负载。

  • 适用场景：当服务器处理请求的时间长短不一时，使用最少连接数算法可以更均衡地分配负载。

3. IP哈希（IP Hash）

  • 描述：根据客户端的 IP 地址进行哈希运算，将请求分配给特定的服务器。这样同一个 IP 地址的请求总是被分配到同一台服务器。

  • 优点：保证了来自同一客户端的请求始终由同一台服务器处理，适合需要会话保持的场景。

  • 适用场景：需要维持会话状态或处理与用户绑定的长期连接时。

4. 权重（Weight）

  • 描述：为每个服务器设置权重，权重越大的服务器接收的请求越多。权重模式可以与轮询或最少连接数模式结合使用。

  • 优点：允许根据服务器的性能差异调整负载分配，性能更好的服务器可以处理更多请求。

  • 适用场景：服务器硬件配置不一致时，可以通过设置权重来优化负载分配。

5. URL哈希（URL Hash

  • 描述：根据请求的 URL 进行哈希运算，将相同 URL 的请求分配到同一台服务器。

  • 优点：可以确保与特定 URL 关联的数据始终存储在同一台服务器上，有助于缓存命中率的提高。

  • 适用场景：在需要高缓存命中率或特定请求需要始终在同一服务器上处理时使用。

三：核心功能-动静分离

动静分离指的是将静态内容和动态内容分开处理的架构策略。其主要目的是静态资源和动态内容分发通过将到不同的服务器或服务上来提高性能和可维护性。
